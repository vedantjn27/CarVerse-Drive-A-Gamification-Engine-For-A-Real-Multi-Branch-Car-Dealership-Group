# CarVerse Drive — Android APK Setup & Release

## Current integration status

The React/Vite frontend has been packaged with Capacitor under `frontend/android/`.
The website remains unchanged: Vercel continues to use `frontend` and the APK uses the same production build, including the Render backend URL.

The original 136 MB Audi R8 source model is deliberately kept in `frontend/local-assets/` and excluded from both Git and the APK. The app uses the optimized R8 asset.

## One-time requirement

Install Android Studio using its standard installer. In the installer, keep the recommended Android SDK, Android SDK Build-Tools, Android SDK Platform, and Android Emulator selections.

## Open and run on a phone

1. Enable **Developer options** and **USB debugging** on an Android phone.
2. Connect the phone by USB and accept its debugging prompt.
3. From `frontend`, run:

   ```powershell
   npm.cmd run android:open
   ```

4. Wait for Android Studio to complete its initial Gradle sync.
5. Select the connected phone in the device selector and press **Run**.
6. Test login, API data, live updates, 3D garage, GLB models, QR flow, audio, PDF export, and responsive layouts.

## Every time frontend code changes

From `frontend`, run:

```powershell
npm.cmd run android:sync
```

Then return to Android Studio and press **Run** again. Do not edit the copied files inside `android/app/src/main/assets/public`; they are regenerated by sync.

## Create a demo APK

In Android Studio:

1. Select **Build → Build Bundle(s) / APK(s) → Build APK(s)**.
2. Choose the generated debug APK from the completion notification.
3. Its usual path is:

   ```text
   frontend/android/app/build/outputs/apk/debug/app-debug.apk
   ```

This APK can be shared directly for a hackathon demo.

## Create a signed release APK

1. Select **Build → Generate Signed Bundle / APK**.
2. Select **APK** and module `app`.
3. Click **Create new** to make a keystore. Store the `.jks` file and passwords securely; never commit them.
4. Select the `release` build type and create the APK.
5. Test the resulting release APK on a physical device before sharing it.

For Google Play publishing, choose **Android App Bundle** instead of APK.

## Safety and rollback

- The pre-Android project snapshot is available locally as Git branch `backup-before-capacitor-android`.
- The website code and Vercel deployment are not replaced or redirected.
- `npm.cmd run build` remains the standard browser production build.
- The APK uses the same HTTPS Render backend configured in `frontend/src/lib/env.ts`.
